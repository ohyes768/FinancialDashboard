## 文档维护规范

### 1. 更新时机
- 代码变更后立即更新相关文档
- 功能新增时更新 PRD 和 TDD
- API 变更时更新 API 文档
- 数据模型变更时更新数据字典

### 2. 更新流程
1. 修改代码实现
2. 更新对应文档
3. 在文档顶部记录变更日期和版本号
4. 提交代码和文档

### 3. 文档质量
- 保持文档的准确性和完整性
- 使用清晰的格式和结构
- 提供充分的示例说明
- 定期审查和更新
